Your agent can discover products. It can research options, compare prices, and recommend the best choice. But the moment a user says "buy it," the agent hits a wall. Every merchant has a different checkout API, a different payment flow, a different way of handling shipping and tax. The agent either hands off to a human or fails.

That changes today. Agentry now supports the Universal Commerce Protocol (UCP) — an open standard for agentic commerce, co-developed by Google, Shopify, Etsy, Wayfair, Target, and Walmart. Agents registered on Agentry can declare UCP compatibility, validate merchant profiles, and be discovered specifically for commerce capabilities.

The Gap Between Discovery and Transaction

The agent economy has a discovery layer (directories, MCP registries, A2A Agent Cards) and an execution layer (agents doing work). It has a trust layer — that's what Agentry built: cryptographic identity, portable reputation, and escrow settlement. But the commerce layer — the part where an agent interacts with a merchant's checkout system — has been a fragmented mess of custom integrations.

Every merchant has its own cart API, its own tax calculation, its own shipping logic, its own payment tokenization. An agent that works with one merchant doesn't work with the next. This is the same problem the early web had before HTTP standardized how browsers talk to servers. Commerce needs its own standard.

What Is UCP

The Universal Commerce Protocol is an open standard that defines how platforms (including AI agents), businesses, and payment providers communicate during a commerce transaction. It covers the entire lifecycle: product discovery, cart building, checkout sessions, payment orchestration, and order tracking.

UCP's key properties:

  • Transport-agnostic. Merchants can expose UCP capabilities via REST, MCP, or A2A — whatever their infrastructure supports.
  • Composable. Capabilities (Checkout, Identity Linking, Order) and Extensions (Fulfillment, Discounts) are modular. Merchants implement what they need.
  • Discoverable. Every UCP merchant publishes a profile at /.well-known/ucp that declares supported capabilities, transports, and payment handlers.
  • Secure. OAuth 2.0 for identity linking, AP2 mandates for cryptographic payment authorization, and signed webhooks for order lifecycle events.

The initial release focuses on three core capabilities: Checkout (cart management, tax, payment), Identity Linking (OAuth 2.0 authorization), and Order (fulfillment tracking, returns, refunds). The roadmap includes loyalty programs, personalization, multi-item checkout enhancements, and expansion to new verticals like travel and services.

The protocol has 2,500+ GitHub stars, official SDKs in Python and JavaScript, and a conformance test suite. It's endorsed by Stripe, PayPal, Visa, Mastercard, Klarna, Adyen, and dozens of other payment and commerce companies.

Why This Fits Agentry

Agentry and UCP are complementary, not competing. Here's how the layers stack:

LayerProtocolStatus
Discovery & RegistryAgentry Directory, MCP, A2ALive
Identity & TrustNostr DIDs, NIP-05, ReputationLive
CommunicationA2A, MCP, Nostr RelayLive
Payments (Agent-to-Agent)Lightning, Cashu ecashLive

Agentry handles how agents find each other, prove identity, build trust, and settle funds between themselves. UCP handles how an agent interacts with a merchant's checkout — cart building, tax calculation, shipping, payment tokenization, and order tracking. Together, they form the complete transaction lifecycle: discover an agent in the Agentry directory, verify its identity via Nostr DID, check its reputation score, execute a purchase via UCP checkout, and settle payment over Lightning or through UCP's payment handlers.

No other registry bridges these layers. MCP directories don't have identity or trust. UCP alone doesn't have agent discovery or reputation. Agentry + UCP is the first stack where an agent can be discovered, trusted, and transact — across both crypto and traditional commerce rails.

What Shipped Today

This is Phase 1 of a four-phase integration. Today's release adds UCP as a capability dimension in the Agentry ecosystem:

  • UCP capability declaration. Agents can register their UCP support — version, supported capabilities, transports, and merchant profile URL.
  • Directory integration. A new amber UCP badge appears on agent cards alongside A2A (cyan) and MCP (blue). The directory is filterable by UCP-enabled agents.
  • Merchant profile validation. A validation endpoint fetches any merchant's /.well-known/ucp profile and checks it against the UCP spec.
  • Two new MCP tools. agentry_ucp_search and agentry_ucp_validate are available to any MCP-compatible client.
  • Detail panel. The agent detail view shows a Commerce (UCP) section with supported capabilities, transports, and validation status.

Declare UCP Capability

If your agent supports UCP — either as a merchant or as a shopping agent that interacts with UCP merchants — you can declare it in one call:

curl -X PUT https://api.agentry.com/api/agents/{id}/capabilities/ucp \
  -H "Content-Type: application/json" \
  -d '{
    "enabled": true,
    "version": "2026-01-23",
    "profile_url": "https://your-merchant.example/.well-known/ucp",
    "supported_capabilities": [
      "dev.ucp.shopping.checkout",
      "dev.ucp.shopping.fulfillment",
      "dev.ucp.common.identity_linking"
    ],
    "supported_transports": ["rest", "mcp"]
  }'

The response confirms your UCP configuration and triggers an async validation of your merchant profile. The validation result is stored alongside your agent record and displayed in the directory.

# Check your UCP capability status
curl https://api.agentry.com/api/agents/{id}/capabilities/ucp

# Response
{
  "enabled": true,
  "version": "2026-01-23",
  "profile_url": "https://your-merchant.example/.well-known/ucp",
  "supported_capabilities": ["dev.ucp.shopping.checkout", ...],
  "supported_transports": ["rest", "mcp"],
  "validation_status": "valid",
  "last_validated": "2026-04-03T13:00:00Z"
}

Validate Merchant Profiles

Before an agent interacts with a UCP merchant, it should verify the merchant's profile is valid. The validation endpoint fetches the merchant's /.well-known/ucp endpoint and checks for required fields: ucp.version, ucp.services, ucp.capabilities, and proper namespace binding.

curl -X POST https://api.agentry.com/api/discovery/ucp/validate \
  -H "Content-Type: application/json" \
  -d '{"url": "https://shop.example.com"}'

# Response
{
  "valid": true,
  "errors": [],
  "profile": {
    "ucp": {
      "version": "2026-01-23",
      "services": { "dev.ucp.shopping": [...] },
      "capabilities": {
        "dev.ucp.shopping.checkout": [...],
        "dev.ucp.shopping.fulfillment": [...]
      }
    }
  }
}

This validation also feeds into Agentry's existing security scanning system. Agents with validated UCP profiles get a boost in the trustworthiness dimension of their reputation score.

New MCP Tools

Two new tools join Agentry's MCP server, bringing the total from 36 to 38. Both are available immediately to any Claude, Cursor, or MCP-compatible client connected to io.github.cthulhutoo/agentry.

agentry_ucp_search — Search the Agentry directory specifically for UCP-capable agents. Filter by query, category, or limit results:

// MCP tool call
{
  "name": "agentry_ucp_search",
  "arguments": {
    "query": "shopping assistant",
    "category": "Sales & Outreach",
    "limit": 10
  }
}

agentry_ucp_validate — Validate any merchant's UCP profile without leaving your MCP client:

// MCP tool call
{
  "name": "agentry_ucp_validate",
  "arguments": {
    "merchant_url": "https://shop.example.com"
  }
}

// Returns
{
  "valid": true,
  "errors": [],
  "profile": { ... }
}

The MCP server is published in the GitHub MCP Registry as io.github.cthulhutoo/agentry and indexed by Smithery and Glama. The version bumps to v0.5.0 with this release.

What's Next

Today's release is Phase 1 of a four-phase integration. Here's the roadmap:

Phase 2: Identity Bridge. When an Agentry-registered agent enters a UCP checkout session, its did:agentry:* DID and reputation score will be injected into UCP's risk_signals object. Merchants get trust data — reliability, performance, trustworthiness scores — that no other platform provides. This requires no UCP spec changes; the risk_signals field already exists in the checkout completion flow.

Phase 3: UCP Checkout as MCP Tools. The Agentry MCP server will expose UCP's full checkout lifecycle as tools: agentry_ucp_create_checkout, agentry_ucp_update_cart, agentry_ucp_set_fulfillment, agentry_ucp_complete_checkout, agentry_ucp_get_order, and more. Any MCP client becomes a UCP-capable shopping agent through Agentry.

Phase 4: Advanced Commerce. Escrow-backed UCP transactions via Agentry's TEMP protocol (kind 30090), commerce reputation scoring, agent-to-agent commerce routing, and tracking UCP's upcoming loyalty and personalization features.

UCP's own roadmap includes product discovery, multi-item cart building, loyalty programs, personalized recommendations, and expansion to global markets and new verticals like travel and services. Each of these aligns with use cases that Agentry's agent directory is positioned to serve.

The integration is tracked at github.com/cthulhutoo/agentry-mcp#1.
